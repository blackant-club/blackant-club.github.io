基于Hexo+GitHub免费搭建个人独立博客

前言：

相信很多小伙伴都有过搭建一个独立博客的想法，独立博客指的是一个拥有完全管理权限的博客。有了个人博客，一切都可以自由定制，自己的地盘自己做主，自由，自在！不必担心像在门户网站博客里那样，只有少数的模板、内容上各种审查以防触雷、管理权限受限等。另外通过独立博客，可以培养自己的独立思考的精神，不盲目做互联网时代的大流附庸。自建博客网站的方式有两种：一是购买服务器、域名，自己搭建，二是利用工具创建免费的博客网站。很显然第一种方式比较费时费钱，而且还要考虑到很多事项，诸如服务器的维护等。而第二种方式无论是从成本还是最终的建设效果来看都很适合于广大的普通用户。网上一堆教程，在此稍微总结一下，写的不对或者有需要的改进的地方请大家指出！Hexo是基于Node.js的一个简洁高效的静态博客框架，可以直接使用Markdown语法来撰写博客。GitHub Pages 是由 GitHub 官方提供的一种免费的静态站点托管服务，虽然搭建有点麻烦，但配置完成后，基本不需要担心维护的事。文章写完后，只需几条命令即可将生成的网页上传到你的github上，然后其他小伙伴就可以看到你的网页，是不是很简单？

开始前的准备工作：

  1. GitHub是全球最大的代码开源平台，因不可抗力因素，国内访问GitHub会时好时坏，最终找到了解决方法：修改hosts文件，而不同操作系统hosts文件的存储位置也不同：

    ​ Windows 系统：C:\Windows\System32\drivers\etc\hosts
    ​ Linux 系统：/etc/hosts
    ​ Mac（苹果电脑）系统：/etc/hosts
    ​ Android（安卓）系统：/system/etc/hosts
    ​ iPhone（iOS）系统：/etc/hosts

  2. 通过：IP查询 在新页面会显示IP地址，复制此ip地址。

  1. 打开 hosts 文件（可用记事本或sublime_text打开），在最后一行添加以下信息：140.82.114.4 github.com （ip地址需要修改成你的！）然后关闭保存。

  1. 修改完成后，在cmd中执行 ipconfig/flushdns 命令即可访问github网站了。（初次打开有点慢，或无法链接，请耐心等待或者按F5刷新一下页面即可。另外此IP地址会不定时更换，若实在打不开GitHub，重复上面1-3步骤）

  1. 如果你有梯子，可以忽略以上步骤，直接进入下面主题：

搭建GitHub Pages：

  1. 注册Github账号并登录

    没有的话可以通过邮箱注册一个，并完成邮件验证。它主要是用来做博客的远程创建、与本地的hexo建立关联。点击下面的链接进行：GitHub注册

  2. 创建仓库

    打开GitHub 主页，点击右边栏的“Start a project”,在Repository name 中输入: 用户名.github.io 比如我的：blackant-clud.github.io（这里的用户名与你注册的用户名是一致的，连在一起就是你博客的域名），Description 选填，勾选 “Initialize this repository with ”下的“Add a README file”，填好后点击 Create repository 创建。

  1. 启用Github Pages

    1. 点击“Settings”进入设置

    1. 点击左边栏的Pages，说明已开启（一般创建完成之后都会默认自动开启）

    1. 选择主题（可忽略，后面有详细介绍）

      1. 点击Choose a theme选择主题

      2. 选择任意一个主题，然后点击select theme

      3. 鼠标下拉，点击Commit changes提交

环境搭建：

  1. 安装Node.js（推荐长期支持版）

    因为Hexo博客框架就是基于Node.js的，所以必须要先安装Node.js环境。

    官网下载地址：Node.js

    安装过程比较简单，推荐默认安装，一路Next即可。搭建过程中还需要使用 npm（Node.js 已带），安装结束后，按Win+R打开命令提示符，输入node -v和npm -v回车，如果出现版本号，说明安装成功了。

  1. 安装Git

    1. Git可将本地的hexo提交到GitHub仓库，建议使用 Git Bash 进行操作

    2. 官网下载地址：Git

    3. 安装过程还是全部默认，注意：走到这一步时，默认勾选的是第一项，需要修改为勾选第二项（从 2020 年 10 月 1 日开始，GitHub 上的所有新库都将用中性词「main」命名，取代原来的「master」，因为后者是一个容易让人联想到奴隶制的术语。），见下图：

    1. 安装完成后，打开Git Bash（因Git没有快捷方式，我们可以从桌面的开始-最近添加中找到，另外也可以在CMD中运行命令），在命令提示符中输入git –version验证是否安装成功。

  1. 安装Hexo

    1. 新建一个文件夹用来存放 Hexo 的程序文件，如 Blog（路径：D:\Blog）。打开Blog文件夹，鼠标右键 Git Bash Here，执行如下命令安装hexo：npm install -g hexo回车

    1. 安装完后输入hexo -v验证是否安装成功。

    1. 目录初始化：

      我的博客文件存放在D:\Blog目录下，cd到Blog里执行hexo init命令，初始化完成后Blog文件夹内会自动生成一些文件：（或者打开Blog文件夹，鼠标右键 Git Bash Here也行，遇到报错，可能因为网络原因，多试几次即可）

    1. 输入npm install安装所需组件

体验Hexo：

  1. hexo generate 生成静态网页（可简写hexo g）

  2. hexo server 启动服务预览（可简写hexo s）

  3. 在浏览器地址栏输入 http://localhost:4000，本地体验Hexo！（ctrl+c可关闭预览）

  1. 打开Themes有多达334款主题任你选择，点击你喜欢的主题名称，即可进入该主题项目，复制地址栏项目链接

    1. 安装主题（比如我选择的是：next主题）：

      定位到D:\Blog，鼠标右键Git Bash Here，输入命令：git clone https://github.com/next-theme/hexo-theme-next.git themes/next（会自动创建并下载到themes下的next文件夹中）

    1. 修改站点配置：

      打开D:\Blog目录下的_config.yml 文件（称之为站点配置文件），下拉找到 theme将默认的landscape，修改为next，关闭保存。

    1. 修改主题配置：

      打开D:\Blog\themes\next下的_config.yml 文件（我们称之为主题配置文件），使用快捷键ctrl+f搜索Scheme Settings，你会发现next主题下有四种样式，默认选择的是Muse，我选择用Pisces，只需将想用的样式前的#号去掉，然后在原来的样式前加个#号，关闭保存即可。

    1. 修改网站设置：

      因为我们clone的别人设置的主题，所以需要修改一些基本信息

    2. 更新主题：

      1. 定位到next，输入cd themes/next回车

      2. git pull（如果报错，可多试几次）

      3. 重新打开D:\Blog文件夹，鼠标右键Git Bash Here，执行hexo g

      4. 执行hexo s

      5. 在浏览器地址栏输入 http://localhost:4000查看新主题。

配置GitHub：

  1. 环境搭建成功后，将你的Git与GitHub帐号绑定，打开Git Bash，依次输入以下命令：git config –global user.name “你的GitHub用户名”和git config –global user.email “你的GitHub注册邮箱” 。如果出现报错，多是因为系统保存了之前其他的用户名（多个github账号），只需执行如下命令：
    git config –global –replace-all user.name “你的 git 的名称” 和 git config –global –replace-all uesr.email “你的 git 的邮箱” ，然后重新执行命令即可

  1. 配置SSH Key

    • 创建SSH密钥：

      输入ssh-keygen -t rsa -C “ryanleacn@outlook.com“（需要换成你的注册邮箱地址），然后一路回车

    • 添加密钥：

      进入C:\Users\用户名.ssh目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容。

    • 登陆 GitHub：

      点击右上角的“人物”图标，进入 Settings 页面，选择左边栏的 SSH and GPG keys，点击右边栏的 New SSH key。Title 随便取个名字，粘贴刚才复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key ，然后输入访问密码即可。

    • 验证连接：

      打开 Git Bash，输入 ssh -T git@github.com回车（固定格式），如果提示Are you sure you want to continue connecting (yes/no)?，输入yes，然后会看到：Hi blackant-club! You’ve successfully authenticated, but GitHub does not provide shell access.看到这个信息说明SSH已配置成功！（设置GitHub密钥原因是：通过非对称加密的公钥与私钥来完成加密，公钥放置在GitHub上，私钥放置在自己的电脑里。GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证用户是否是合法用户，为了省去每次输入密码的步骤，采用了ssh，当你推送的时候，git就会匹配你的私钥和GitHub上面的公钥是否配对，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是合法的）

  2. 修改站点配置（_config.yml）

    • 打开D:\Blog下的_config.yml文件（站点配置文件），下拉到底部，将原来的：

    • 修改成：（注意：冒号后面有空格！格式为：git@github.com:blackant-club/blackant-club.github.io.git，关闭保存。

    • 有的朋友可能找不到这个git@xxxxxx，只需点击网站左上角你的仓库，然后会跳转到新的页面，按照下图顺序操作即可复制

      链接，回到上步ctrl+v粘贴即可。

部署博客：（推荐使用Typora）

  1. Typora官网下载地址：Typora

    安装过程比较简单这里就不赘述了，另外需要学习Markdown语法，大家也不用担心它有多难，估计有10分钟就会学会，剩下的就看熟练度了。

  2. 打开D:\Blog文件夹，鼠标右键选择Git Bash Here，输入命令：hexo new ‘my_first_blog’（博客名自己随便起，用这个命令的好处是帮我们自动生成了时间。），hexo会自动在_posts下生成一个my-first-blog.md文件，打开这个.md文件，利用Typora编辑器按markdown语法就可以写博客了。

  1. 博客写好后，可执行下面3个命令：

    1. hexo clean 清除缓存，网页正常的情况下可忽略此命令

    2. hexo generate 生成静态网页（可简写hexo g）

    3. hexo server 启动服务预览（可简写hexo s，ctrl+c可关闭预览）

    4. 在浏览器地址栏输入http://localhost:4000回车，可以在本地预览网站的初步样子。

部署到GitHub上：

  1. 安装git插件：打开D:\Blog文件夹，鼠标右键选择Git Bash Here，输入命令: npm install hexo-deployer-git –save

  2. 本地项目部署到GitHub上：

    依次执行下面3个命令：

    1. hexo clean

    2. hexo g

    3. hexo d

    4. 按F5刷新Github网页，点击左上角你的仓库，你会发现本地项目已上传到GitHub上了

  3. 在浏览器地址栏输入你博客网站的网址如：https://blackant-club.github.io/即可访问你的博客了！但遗憾的是博文中无法识别图片，图片存在本地，上传到博客网站就无法显示了，就算一个图一个图的复制粘贴上去，想移植到其他博客网站，图片也会失效，所以我们需要图床。

搭建Typora的图床：

那么图床是干什么的？简单的说图床就是一个便于在博文中插入在线图片链接的个人图片仓库。虽然可以利用GitHub搭建自己免费的图床，但经测试实在是太不稳定了，所以推荐用国内的 Gitee 来搭建。Gitee和GitHub都是代码托管平台，和GitHub相比最大的优点就是其访问速度。

  1. 注册一个Gitee 账号，就能拥有一个免费的图床，传送门：Gitee

  2. 新建仓库的要点：

    • 设置仓库名称

    • 将仓库设为公开

    • 勾选使用Readme文件初始化这个仓库，这样Gitee（码云）会自动给你的仓库建立master分支，这点很重要!!! ，因为使用github做图床PicGo好像会自动帮你生成master分支，而PicGo里的gitee插件不会帮你自动生成分支。

    • 点击创建，如图：

  3. 图床搭建好了，推荐使用一个开源图床工具 PicGo 来作为我们的图片上传工具。PicGo是一款简易的图床上传工具，可以通过拖拽或者复制粘贴的方式将图片上传到图床。软件安装比较简单这里略过。（注意：安装软件时选择仅为我安装，不要安装在C盘上，否则后面上传图片总是失败，因为这个问题找了半天才发现是路径的问题！）

  4. 配置PicGo：

    1. 登录Gitee，点击右上角圆形图标->设置->私人令牌->生成新令牌

    2. 填写令牌描述信息及勾选相关信息，然后点击提交即可，提交后会输入密码验证生成。

    3. 注意令牌只会显示一次，所以你最好把它复制下来到你的备忘录存好，方便下次使用，否则下次有需要重新新建。

    4. PicGo默认是没有Gitee图床的，但我们可以通过安装插件来实现。打开插件设置，在搜索框中搜索gitee-uploader插件，然后点击安装，当然也可以安装github-plus或者gitee

    5. 点击图床设置，下拉找到并点击gitee，在右边栏进行gitee设置：

      • repo：就是刚才创建的用户名/仓库名

      • branch：默认为master即可

      • token：刚才生成的令牌

      • path：可自定义，但是要填，否则可能导致上传失败

      • customPath：图片归档方式，以年、年月……（可填可不填）

      • customUrl：保持默认

  5. 配置Typora：

    1. 打开Typora，点击文件->偏好设置->图像->按照图中勾选->关闭保存，之后，只要我们在typora中粘贴一张图片，就会自动上传到我们的gitee仓库中，是不是非常的方便。

    2. Typora中插入图片有两种方式：

      • 利用快捷键将图片插入博文中：ctrl+shift+i或者直接键入! + [] +()
      • 通过上传区上传，ctrl+c复制图片，ctrl+v粘贴到博文中或者将图片拖拽进来都可以

    3. 上传成功的图片在相册中显示，可以在这里进行删除，删除的同时gitee库中的图片会一并删除（备注：当图片上传到图片库后，即使你删除了本地图片文件夹，在Typora中插入的图片以及部署到Github上的博文中的图片依然可见）

    4. gitee缺陷：在后续的使用过程中，大概率会碰到，有时候无法访问图片的情况，但是通过图片的地址，是可以访问到图片的，这是因为gitee为了出于安全考虑，大于1M的文件，需要登录之后才可以访问。

重新部署到GitHub上：

经过上面一系列的设置，下面将本地文件重新部署到GitHub上。打开D:\Blog文件夹，鼠标右键选择Git Bash Here，可依次执行如下命令：

  1. hexo clean

  2. hexo g（如果不想预览，可省略第3和第4步，直接跳到第5步）

  3. hexo s

  4. 在浏览器地址栏输入http://localhost:4000回车，进行本地预览。

  5. hexo d

  6. 浏览器地址栏输入博客网址：https://blackant-club.github.io/，如图所示，图片已显示！（注意：如果在手机上操作，请复制链接在浏览器中打开，因为在微信中直接打开，可能出现部分图片不显示的问题）

至此基于Hexo+GitHub搭建个人独立博客过程结束。目前来看，博客还相当简陋，下一步想进行一些美化，我很期待！希望这篇文章能给你一点点灵感与想法！
