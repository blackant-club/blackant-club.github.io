基于PicGo+GitHub搭建免费图床

发表于 2021-05-22 更新于 2021-05-25 阅读次数：
本文字数： 1.8k 阅读时长 ≈ 2 分钟

在写第二篇博文的时候，有个章节介绍了如何利用PicGo+Gitee搭建免费图床，本想可以一劳永逸，但是在上传第50张图片时，出现报错，提示You can only upload 50 Images per week!（你一周只能上传50张图片），当时看了一下时间是周五，要等到下个周一才能继续上传，这谁受得了！于是萌生用PicGo+GitHub来搭建第二款免费图床的想法，不知是否可行，接下来让我们一起试试吧！

配置GitHub：

  1. 注册GitHub账号并登录，注册过程可参考我的博客蚂蚁小黑中的第一篇文章my_first_blog，这里略过。

  2. 创建一个新仓库用于存放图片。

  3. 如图自己起个仓库名，勾选public（必须的）和Add a README file，点击Create repository

  4. 见下图，按照顺序点击：

  5. 点击左边栏的Personal access tokens，再点击右边栏的Generate new token

  6. 输入GitHub账号密码，填写Note，勾选repo，点击Generate token

  7. 注意：生成的token只会在这里显示一次，所以记得单独保存下来哦！！！，至此，Github的配置完成。

配置PicGo：

  1. 下载安装PicGo，选择picGo-Setup-XXX.exe下载软件安装包即可。注意：安装软件时选择仅为我安装，不要安装在C盘上！！！，否则后面上传图片总是失败，因为这个问题找了半天才发现是路径的问题！）

  2. 打开PicGo，点击左边栏的图床设置-GitHub图床进行设置：

    • 仓库名：[github用户名]/[新建的仓库名称]
    • 分支：2020.10.01开始，github的默认分支名变更为main
    • 设定token：第一步创建的token
    • 指定存储路径patch：可填可不填，如果想将图片上传到仓库的指定目录下，可以填写目录名加/，例如我设置的为img/
    • 设定自定义域名：这里统一用jsdelivr的CDN加速域名，https://cdn.jsdelivr.net/gh/[github用户名]/[仓库名]@main ，注意，此处的分支一定要填写@main，否则默认使用master分支。而现在github创建的默认分支名为main，如果不指定，则会出现图片不能上传的情况。
    • 至此，github+picGo的配置完成，可以在上传区进行图片上传了。

  3. 对于之前已经安装过PicGo的朋友，不需要重新安装，可打开PicGo点击图床设置，点击GitHub图床（之前选择的是gitee），然后按照上一步配置，见下图：

  4. 关闭窗口，重新启动PicGo，此时PicGo已将原来的图床Gitee替换为GitHub图床，见下图：

  5. 此时可以上传图片了，我的使用方法是：打开Typora，在文章中需要插入图片的地方，按快捷键ctrl+shift+i，然后选中要插入的图片即可上传。（此操作下：如果之前没打开PicGo，此时会自动打开，但不需要在PicGo中做任何操作，前提是需要配置好Typora）

  6. 配置完PicGo后却上传图片失败，可以参考以下方法解决：

    • 检查自定义域名是否正确

    • 仓库名不要有空格

    • 图片名字不要带有特殊符号，如：%、+、*、空格等

    • 建议开启时间戳重命名，防止图片名字重复

    • PicGo应用不稳定因素，需重启应用

    • 想删除不想要的图片（已上传完毕的图片），可点击PicGo左边栏的”相册“，最近上传的照片都在这里显示，不想要的照片，可通过点击”垃圾桶“图标进行删除，会一并删除保存在GitHub仓库中的这张图。

配置Typora：

关于配置Typora这里就略过，大家可在蚂蚁小黑中的第一篇文章my_first_blog中查看。至此所有配置完成（以后可以在图床设置中随意切换GitHub图床和Gitee图床使用），大家可以愉快的玩耍了！

备忘：

  • 查看所有已安装的 hexo 插件：npm ls -dept 0
  • 安装：npm install 插件名 –save （注意save前是两个-）
  • 卸载：npm uninstall 插件名 –save （注意save前是两个-）
  • 创建文章：hexo new “文章名”
  • 清理缓存：hexo clean
  • 创建静态网页：hexo g
  • 启动预览：hexo s
  • 关闭预览：ctrl+c
  • 上传服务器：hexo d
  • 创建静态网页并上传服务器：hexo d -g
